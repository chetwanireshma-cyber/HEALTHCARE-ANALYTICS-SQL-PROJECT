# HEALTHCARE-ANALYTICS-SQL-PROJECT

📌 Project Overview
This project focuses on analyzing healthcare data to extract meaningful insights related to hospital revenue, patient distribution, department performance, and doctor-level analytics using SQL.
The dataset includes information about patients, doctors, appointments, and departments.

🎯 Objectives
Calculate total hospital revenue
Analyze department-wise revenue performance
Identify top-performing doctors based on treatment revenue
Analyze city-wise patient distribution
Generate monthly revenue trends
Extract business insights for healthcare decision-making.

🗂 Database Structure
The project consists of the following tables:
Patients
Doctors
Appointments
Departments
Relationships were established using primary and foreign keys.

🛠 Tools & Technologies Used
MySQL
MySQL Workbench
SQL (Joins, Group By, Having, Aggregate Functions, Date Functions)

📊 Key SQL Concepts Applied
INNER JOIN for multi-table analysis
GROUP BY and HAVING for aggregation
SUM, COUNT, AVG for revenue and patient metrics
ORDER BY for ranking and sorting
MONTH() function for time-based analysis

📈 Sample Insights Generated
Total hospital revenue analysis
Cardiology identified as highest revenue-generating department
Top revenue-generating doctor identified
Delhi had highest patient count
Monthly revenue trends visualized

💡 Business Impact
This project demonstrates how SQL can be used to transform raw healthcare data into actionable insights that support hospital management decisions and performance monitoring.
